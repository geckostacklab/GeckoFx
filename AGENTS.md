# AGENTS.md

## Package manager

Use **Bun** (not npm/yarn/pnpm). Lockfile is `bun.lock`.

## Commands

```sh
bun dev              # Vite dev server
bun run build        # tsc -b (typecheck all project refs) then vite build
bun run lint         # ESLint (flat config)
bun run preview      # Vite preview of dist/
bun run generate:docs  # Re-generate src/routes/docs/*.lazy.tsx from component.map.ts
```

Build must pass both typecheck and lint. There are no tests yet.

## Adding a new SVG component

1. Create the SVG React component in `src/components/svg/YourName.tsx`
2. Add an entry to the array in `src/component.map.ts`
3. Run `bun run generate:docs` — this creates the lazy route file at `src/routes/docs/your-name.lazy.tsx`
4. The lazy route and `src/routeTree.gen.ts` (auto-generated by TanStack Router) are both checked into git

## Architecture

- **Single SPA, no backend.** React 19 + Vite 8 + TanStack Router + Tailwind CSS v4.
- **Entry point:** `index.html` → `src/main.tsx`
- **Routes:** file-based under `src/routes/`, code-split via TanStack Router. `/` is the landing page, `/docs/*` are per-component docs pages, `/test` is a manual testing sandbox.
- **SVG components** live in `src/components/svg/`. They are hand-written JSX SVGs (not `.svg` imports), animated with the `motion` library (imported from `motion/react`, not `framer-motion`).
- **UI components** are in `src/components/ui/` (layout wrapper, buttons, tabs for docs).
- **Component metadata** lives in `src/component.map.ts` — consumed by both the docs site and the code generator script.

## Key quirks

- **Tailwind CSS v4** with custom theme tokens and utilities defined in `src/index.css` (e.g., `bg-hatching`, `screen-line-*`, `scrollbar-none`).
- **React Compiler** is enabled via `babel-plugin-react-compiler` (configured in `vite.config.ts`).
- **Importing raw source:** lazy route files use Vite's `?raw` import suffix to embed SVG component source as a string for the Code tab.
- **Dark-mode only.** Design is built on a single dark theme.
- **Generated files checked in:** `src/routeTree.gen.ts` and `src/routes/docs/*.lazy.tsx` are checked into git. They are regenerated automatically during dev/build (route tree) or manually (docs routes).
